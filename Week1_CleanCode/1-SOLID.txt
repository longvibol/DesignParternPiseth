==> SOLID is the basic of design pattern 

SOLID is a set of 5 object-oriented design principles introduced by Robert C. Martin.

គោលដៅសំខាន់គឺ៖

The goal is to create code that is:

• Easy to understand
• Easy to change
• Easy to test
• Flexible
• Maintainable


1- SRP : Single Responsibility Principle : ទំនួលខុសត្រូវតែមួយ (មានហេតុផងតែមួយទេ)

class នឹងគួរតែមានហេតុផលតែមួយទេ ដើម្បីធ្វើការផ្លាស់ប្តូរ 

A class should have only one reason to change

Ex: Print() when we need print we go to change this class 

វាអាចធ្វើការបានច្រើនតែ ក្នុងហេតុផលតែមួយ

2- O : Open Closed Principle (OCP)
-  បើអោយគេ extension (បើ​ អោយគេបន្ថែម feature បាន)
- មិនមិនអោយគេធ្វើការកែប្រែ

យើងបន្ថែម new behavior ហើយវាមិនទៅកែកូដចាស់ You should add new behavior without changing existing code.

Example:

public BigDecimal calculateDiscount(String customerType, BigDecimal amount) {
		if ("SILVER".equals(customerType)) {
			return amount.multiply(BigDecimal.valueOf(0.05));
		}
		if ("GOLD".equals(customerType)) {
			return amount.multiply(BigDecimal.valueOf(0.10));
		}
		if ("PLATINUM".equals(customerType)) {
			return amount.multiply(BigDecimal.valueOf(0.15));
		}
		return BigDecimal.ZERO;
	}

ចឹងដើម្បីដោយស្រាយបញ្ហានឹងគេបង្កើត pattern មកមានដូចជា៖

4- I = Interface Segregation Principle (ISP)

Clients should not depend on methods they don't use. Clientsមិនគួរពឹងផ្អែកទៅលើ Method ដែលវាអត់ប្រើនោះទេ

បើមាន method ណាអត់ខុស មិនចាំបាច់វាមកទេ

Bad Example
public interface Worker {
	void work();
	void eat();
	void sleep();
	void drive(); // this one is not use with Worker
}

No need to put driver in the work 

ចឹងយើងអាចបំបែកដូចនឹងផ្សេងគ្នាទៅ​ ចាំយើងធ្វើការ Implement វា

Better Design
interface Worker {
void work();
}
interface Driver {
void drive();
}
interface Eater {
void eat();
}
Small focused interfaces.
