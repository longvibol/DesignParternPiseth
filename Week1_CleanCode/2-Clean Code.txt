3.3 Single Responsibility Principle (SRP) ទំនួលខុសត្រូវតែមួយ

Bad:

public class UserService {

	public void register(User user) {
		validate(user);
		saveToDatabase(user);
		sendEmail(user);
		generateReport(user);
	}
}

នៅក្នុង UserService វាមានការទទួលខុសត្រូវច្រើន ដោយសារតែនៅក្នុង 

validate(user); និងផ្សេងទៀត នឹង គេអាចធ្វើការ ហៅវាមកប្រើបានច្រើន បន្ទាប់ពីបង្កើត Object UserService ហើយ ចឹងវាមានតួនាទីច្រើនពេក

==> Better Design 

យើងធ្វើការបង្កើត​ class ផ្សេងសម្រាប់ each function ហើយធ្វើការហៅវាយក មកប្រើ

Better:
public class UserService {
	private final UserValidator validator;
	private final UserRepository repository;
	private final EmailService emailService;


	public void register(User user) {
		validator.validate(user);
		repository.save(user);
	}
}

- private final UserValidator validator; វាជា  class ថ្មីដែលយើងបានបង្កើត ហើយហៅវាយកមកប្រើនៅក្នុង UserService 
ចឹងយើងធ្វើការបង្កើតវា  component ផ្សេងៗគ្នា


3-4 Avoid Duplicated Code យើងសរសេរ កូដ ដដែរៗ

Validate to the user age: 

if (user.getAge() < 18) {
throw new IllegalArgumentException();
}
if (customer.getAge() < 18) {
throw new IllegalArgumentException();
}

- it validate to the age of the user and customer (these both are the same valiation on the age) 
==> we should create only one validation on the age 

Good:
private void validateAdult(int age) {
if (age < 18) {

}
}
