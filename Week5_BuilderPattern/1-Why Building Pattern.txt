Lesson 2

Telescoping Constructor Pattern (វាពង្រេចរហូត បន្ទាប់ពីបង្កើត Object ហើយ)

- បន្ទាប់នឹងចាំបាច់ត្រូវការតែលេខកូដ បន្ទូប់ទេ Suppose only roomCode is required.
- បន្ធាប់មកគេត្រូវការបន្ថែម" price "​ then " currency " then "wifi"

==> Developer ភាគច្រើនគាត់បង្កើត optional Constructor (overall constructor) 

1-public Room(String roomCode)

2-

public Room(
String roomCode,
BigDecimal price,
Currency currency)

3- 

public Room(
String roomCode,
BigDecimal price,
Currency currency,
boolean wifi)

This is called : Telescoping Constructor ហើយអា Optional fields នឹងវាមានហូរហែរ 

why it is bad:

- Constructor ច្រើនពេក
- Hard maintance 
- ពិបាកចាំ
- impossible after 15 fields

Constructor មានច្រើនប្រភេទ៖
- default Constructor 
- NoArg Constructor 
- Constructor with parameter
